本文档涵盖 Pi 各核心 Package 的角色与依赖关系、开发 Agent 需要使用的关键接口、以及四种典型 Agent 的完整实现方案。

1. 包架构概览

Pi 是一个 monorepo，核心包共 5 个，按抽象层次从低到高排列：

packages/
├── ai/          @earendil-works/pi-ai          统一 LLM API 层
├── agent/       @earendil-works/pi-agent-core  通用 Agent 框架
├── coding-agent/@earendil-works/pi-coding-agent完整编程 Agent (CLI+TUI+SDK)
├── tui/         @earendil-works/pi-tui         终端 UI 库 (差分渲染)
└── server/      @earendil-works/pi-server      实验性 RPC 服务

1.1 依赖关系

pi-ai              ← 无外部依赖，统一 LLM Provider 调用
pi-agent-core      ← 依赖 pi-ai，提供 Agent 循环 + 工具执行框架
pi-coding-agent    ← 依赖 pi-ai + pi-agent-core + pi-tui，全功能 CLI + SDK
pi-server          ← 依赖 pi-coding-agent，RPC over HTTP

1.2 各 Package 核心模块

pi-ai (@earendil-works/pi-ai)

统一调用 LLM Provider，包装 Anthropic、OpenAI、Bedrock、Google 等各种 API。

模块作用
src/types.ts核心类型：Model<T>、Context、Message、AssistantMessage、Tool、StreamOptions
src/models.tsModels 接口 + Provider 接口 + createModels() + createProvider()
src/models-store.tsModelsStore / InMemoryModelsStore，持久化模型目录
src/api/*.ts各 API 实现（openai-completions、anthropic-messages 等）
src/api/lazy.tslazyStream() / lazyApi() — 延迟加载 API 模块
src/providers/*.ts各 Provider 工厂函数
src/auth/*.ts认证系统（OAuth、API Key、Credentials）
compat.ts向后兼容入口，暴露 getModel() 和 streamSimple()

pi-agent-core (@earendil-works/pi-agent-core)

Agent 循环 + 工具执行框架，不依赖具体 LLM Provider。

模块作用
src/agent.tsAgent 类 — 状态管理、事件、消息队列、steering/follow-up
src/agent-loop.tsrunAgentLoop() — 底层循环（LLM 调用 → 工具执行 → 循环/停止）
src/types.tsAgentTool、AgentState、AgentEvent、AgentContext、AgentMessage、StreamFn
src/stream-fn.tsStreamFn 默认值设置
src/harness/agent-harness.tsAgentHarness — 会话管理、compaction、skills、prompt templates
src/harness/tools/内置工具实现（read、edit、bash、write、grep）
src/harness/session/会话持久化（JsonlRepo、MemoryRepo、JsonlStorage）

pi-coding-agent (@earendil-works/pi-coding-agent)

完整编程 Agent，将 pi-ai + pi-agent-core 组合成可用的 CLI 和 SDK。

模块作用
src/core/sdk.tscreateAgentSession() 一站式入口
src/core/agent-session.tsAgentSession 类 — 高级会话接口
src/core/agent-session-runtime.tsAgentSessionRuntime — 运行时生命周期
src/core/model-runtime.tsModelRuntime — 模型/Provider/认证管理
src/core/settings-manager.tsSettingsManager — 配置管理
src/core/session-manager.tsSessionManager — 会话持久化
src/core/extensions/扩展系统（ExtensionRunner、defineTool()）
examples/sdk/13 个 SDK 示例（01-minimal.ts ~ 13-session-runtime.ts）
examples/extensions/90+ 扩展示例

2. 开发 Agent 所需 Package 与关键接口

2.1 方案 A：使用 pi-coding-agent SDK（推荐）

只需要引入一个 Package：

import {
  createAgentSession,       // 一键创建带 session 的 Agent
  ModelRuntime,             // 管理模型 Provider、认证、列表刷新
  SessionManager,           // 会话持久化（jsonl 文件或内存）
  SettingsManager,          // 管理 compaction、retry 等设置
  defineTool,               // 定义自定义工具
  type ToolDefinition,      // 工具定义类型
  type ResourceLoader,      // 加载 skills、extensions、prompts
  type AgentSessionEvent,   // 会话事件类型
} from "@earendil-works/pi-coding-agent";
接口/类用途
createAgentSession()配置 model、tools、cwd、settings 后返回 AgentSession
ModelRuntime管理 Provider 注册、API Key 注入、模型刷新
SessionManager控制会话持久化行为（文件系统或内存）
SettingsManager配置 compaction、retry 策略等
defineTool()以类型安全方式定义自定义工具
ResourceLoader资源加载器，可完全自定义或使用默认加载器

2.2 方案 B：使用 pi-agent-core（轻量，自定义 Provider）

import {
  Agent,                    // Agent 类 — 状态、事件、队列
  type AgentTool,           // 工具定义（低层接口）
  type AgentEvent,          // Agent 事件
  type AgentState,          // Agent 状态
  type StreamFn,            // 流函数类型
  type ThinkingLevel,       // 思考级别
} from "@earendil-works/pi-agent-core";

// 需要自行提供 StreamFn，可以来自 pi-ai：
import { createModels } from "@earendil-works/pi-ai";
// 或使用 compat API
import { getModel, streamSimple } from "@earendil-works/pi-ai/compat";

2.3 方案 C：纯 pi-ai（只有 LLM 调用，无工具）

import { createModels, createProvider, type Models, type Provider } from "@earendil-works/pi-ai";
// 或 compat API（更简洁）：
import { getModel, streamSimple, completeSimple } from "@earendil-works/pi-ai/compat";

3. 四种 Agent 实现方案

3.1 纯 LLM 调用

无工具，无循环，直接调用 LLM。适合问答、文本生成。

import { getModel, streamSimple } from "@earendil-works/pi-ai/compat";

const model = getModel("anthropic", "claude-sonnet-4-5");
if (!model) throw new Error("Model not found");

const stream = streamSimple(model, {
  messages: [{ role: "user", content: "用一句中文说 hello", timestamp: Date.now() }],
});

for await (const event of stream) {
  if (event.type === "text_delta") process.stdout.write(event.delta);
}
const response = await stream.result();
console.log("\n使用量:", response.usage);

如果用新 API（createModels()）：

import { createModels } from "@earendil-works/pi-ai";
import { anthropicProvider } from "@earendil-works/pi-ai/providers/anthropic";

const models = createModels();
models.setProvider(anthropicProvider());
models.setRuntimeApiKey("anthropic", process.env.ANTHROPIC_API_KEY);

const model = models.getModel("anthropic", "claude-sonnet-4-5");
const context = {
  messages: [{ role: "user", content: "Hello", timestamp: Date.now() }],
};
const msg = await models.completeSimple(model, context);
console.log(msg.content);

3.2 告警分析 Agent

只读工具 + 自定义数据源工具，无写入权限。适合自动化运维场景。

import { createAgentSession, SessionManager } from "@earendil-works/pi-coding-agent";
import { getModel } from "@earendil-works/pi-ai/compat";
import { defineTool } from "@earendil-works/pi-coding-agent";

// 自定义工具：获取告警数据
const fetchAlertsTool = defineTool({
  name: "fetch_alerts",
  label: "Fetch Alerts",
  description: "获取最近 N 条告警数据",
  parameters: {
    type: "object",
    properties: {
      count: { type: "number" },
      severity: { type: "string", enum: ["critical", "warning", "info"] },
    },
    required: ["count"],
  },
  execute: async (params) => {
    const alerts = [
      { severity: "critical", message: "CPU 98%", source: "web-01" },
      { severity: "warning", message: "磁盘剩余 12%", source: "db-02" },
    ];
    return {
      content: [{ type: "text", text: JSON.stringify(alerts, null, 2) }],
      details: { count: alerts.length },
    };
  },
});

// 自定义工具：查询运行手册
const lookupRunbookTool = defineTool({
  name: "lookup_runbook",
  label: "Lookup Runbook",
  description: "根据告警类型查找处理手册",
  parameters: {
    type: "object",
    properties: { alertType: { type: "string" } },
    required: ["alertType"],
  },
  execute: async (params) => {
    const runbooks: Record<string, string> = {
      high_cpu: "1. 检查活跃进程\n2. 查看最近部署",
      disk_full: "1. 清理日志\n2. 扩容磁盘",
    };
    return {
      content: [{ type: "text", text: runbooks[params.alertType] ?? "未找到手册" }],
      details: {},
    };
  },
});

const model = getModel("anthropic", "claude-sonnet-4-5")!;
const { session } = await createAgentSession({
  model,
  tools: ["read", "grep"],  // 只读 — 安全
  customTools: [fetchAlertsTool, lookupRunbookTool],
  sessionManager: SessionManager.inMemory(),
});

session.subscribe((event) => {
  if (event.type === "message_update" && event.assistantMessageEvent.type === "text_delta")
    process.stdout.write(event.assistantMessageEvent.delta);
});

await session.prompt("分析 web-01 上的告警并给出处理建议");
session.dispose();

3.3 自动问答机器人 Agent

使用 Agent 类（最小依赖），利用 steer() / continue() 实现多轮对话。适合客服、内部支持。

import { Agent } from "@earendil-works/pi-agent-core";
import { getModel, streamSimple } from "@earendil-works/pi-ai/compat";

const searchTool = {
  name: "search_knowledge_base",
  label: "Search KB",
  description: "在知识库中搜索相关内容",
  parameters: {
    type: "object",
    properties: { query: { type: "string" }, topK: { type: "number" } },
    required: ["query"],
  },
  execute: async (_id, params) => {
    const results = [
      { title: "如何重置密码", url: "https://wiki.example.com/reset" },
    ].filter(r => r.title.includes(params.query));
    return {
      content: [{ type: "text", text: JSON.stringify(results.slice(0, params.topK ?? 2)) }],
      details: {},
    };
  },
};

const agent = new Agent({
  streamFn: streamSimple,
  initialState: {
    systemPrompt: "你是内部技术支持。简洁、步骤清晰。",
    model: getModel("anthropic", "claude-sonnet-4-5")!,
    tools: [searchTool],
  },
  // 审计工具调用
  beforeToolCall: async (ctx) => {
    console.log(`[AUDIT] ${ctx.toolCall.name}`, ctx.args);
  },
});

agent.subscribe((event) => {
  if (event.type === "message_update" && event.assistantMessageEvent.type === "text_delta")
    process.stdout.write(event.assistantMessageEvent.delta);
});

// 多轮对话
await agent.prompt("我忘记密码了，怎么重置？");
await agent.waitForIdle();

agent.steer({ role: "user", content: [{ type: "text", text: "重置后需要重新登录 VPN 吗？" }], timestamp: Date.now() });
await agent.continue();
await agent.waitForIdle();

3.4 代码审查 Agent

最复杂的场景：访问 Git diff、运行 linter、读写文件、按标准审查。需要自定义工具 + 内建读写工具的组合。

import { createAgentSession, SessionManager, defineTool } from "@earendil-works/pi-coding-agent";
import { getModel } from "@earendil-works/pi-ai/compat";
import { execSync } from "node:child_process";

// 1. Git Diff 工具
const gitDiffTool = defineTool({
  name: "git_diff",
  label: "Git Diff",
  description: "获取当前分支与目标分支的代码变更",
  parameters: {
    type: "object",
    properties: {
      targetBranch: { type: "string", default: "main" },
      path: { type: "string" },
    },
  },
  execute: async (params) => {
    const target = params.targetBranch ?? "main";
    const diff = execSync(`git diff ${target}...HEAD${params.path ? ` -- ${params.path}` : ""}`, {
      encoding: "utf-8", maxBuffer: 10 * 1024 * 1024,
    });
    return { content: [{ type: "text", text: diff.slice(0, 50000) }], details: {} };
  },
});

// 2. Lint 检查工具
const lintCheckTool = defineTool({
  name: "lint_check",
  label: "Lint Check",
  description: "对指定文件运行 linter",
  parameters: {
    type: "object",
    properties: {
      filePath: { type: "string" },
      linter: { type: "string", enum: ["biome", "eslint"], default: "biome" },
    },
    required: ["filePath"],
  },
  execute: async (params) => {
    try {
      const output = execSync(`npx ${params.linter} ${params.filePath} 2>&1`, { encoding: "utf-8" });
      return { content: [{ type: "text", text: output || "无 lint 问题" }], details: {} };
    } catch (e) {
      return { content: [{ type: "text", text: String(e.stderr || e.stdout || e) }], details: {} };
    }
  },
});

// 3. 审查标准规则
const standardsTool = defineTool({
  name: "get_review_standards",
  label: "Review Standards",
  description: "获取项目的代码审查标准",
  parameters: { type: "object", properties: {} },
  execute: async () => ({
    content: [{
      type: "text",
      text: [
        "- 优先用 interface 而非 type",
        "- 函数不超过 50 行",
        "- 所有公共 API 必须有 JSDoc",
        "- 严禁 any（除非注释说明）",
      ].join("\n"),
    }],
    details: {},
  }),
});

const model = getModel("anthropic", "claude-sonnet-4-5")!;
const { session } = await createAgentSession({
  model,
  cwd: process.cwd(),
  tools: ["read", "grep", "edit", "write"],
  customTools: [gitDiffTool, lintCheckTool, standardsTool],
  sessionManager: SessionManager.inMemory(),
});

session.subscribe((event) => {
  if (event.type === "message_update" && event.assistantMessageEvent.type === "text_delta")
    process.stdout.write(event.assistantMessageEvent.delta);
});

await session.prompt(`请审查当前分支的变更：
1. 用 git_diff 获取变更
2. 用 get_review_standards 获取标准
3. 逐文件审查
4. 输出审查纪要`);

session.dispose();

4. 选型指南

场景推荐 Package核心 API
纯 LLM 调用（无工具）pi-aistreamSimple() / completeSimple()
简单 Agent + 自定义工具pi-agent-coreAgent + AgentTool + streamFn
全功能编程 Agentpi-coding-agent (SDK)createAgentSession() + defineTool()
嵌入式/服务端 Agentpi-agent-core + createModels()Agent + Models.streamSimple()
告警分析 Agentpi-coding-agentdefineTool() + 只读内建工具
问答机器人pi-agent-coreAgent + steer()/continue()
代码审查 Agentpi-coding-agentdefineTool() + 内建读写工具

关键设计原则

  • 抽象分层：pi-ai → pi-agent-core → pi-coding-agent 逐层封装，下层不依赖上层。
  • 工具即接口：自定义工具通过 defineTool() 或 AgentTool 接口注册，Agent 循环自动调度。
  • 安全边界：Tools 参数控制 Agent 能访问的资源，配合 beforeToolCall 实现细粒度权限。
  • 事件驱动：subscribe() 暴露完整生命周期事件，适合 UI 渲染和日志记录。
  • 多轮对话：steer()（运行中注入）+ followUp()（结束后注入）覆盖所有对话流程。
  • 配置弹性：从 createAgentSession() 一键启动到 Agent 类完全自定义，适应不同需求。
